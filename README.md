# 創意軌道規劃 Creative Track Planner

> 幫學生跨過「軌道規劃」這一關的線上 AI 輔助工具。

🔗 **線上使用：** https://sanwanh.github.io/creative-track/
📄 **競賽規則：** [創意軌道 競賽規則（Google Drive）](https://drive.google.com/file/d/1bY1eLhCWXWmU1_3yAhvY_0QL5YWVXzw8/view)

## 為什麼做這個

我以前在帶「創意軌道」這個機器人競賽。教的過程中發現一件事：**真正卡住學生的往往不是動手拼，而是「規劃」這一步**——手上就那幾片板子要整場循環使用，還得繞過所有加分點、最後超過終點線，很多學生光是想「到底要怎麼擺才走得到終點」就卡很久，甚至排不出可行的路線，挫折感很重。

所以我做了這個網站，用一套**自動規劃演算法（AI）**，幫學生在幾秒內找出一條可行、而且組數盡量少的擺法。它的定位不是「幫學生作答」，而是：

- 讓學生**先看到一個可行解**，理解「原來可以這樣繞」，建立空間規劃的直覺；
- 把卡關的時間省下來，讓學生把心力放在**理解規則、嘗試與創意**上；
- 老師備課、出題、驗證題目「有沒有解」時也能快速確認。

一句話：**降低規劃門檻，讓更多學生願意繼續玩下去。**

---

## 這個工具能做什麼

- 🎲 **自動規劃**：給定抽到的板子與加分點，用 A\* 搜尋找出「組數最少」的完整路線，並在場地上畫出來。
- ✍️ **抽籤 / 手動輸入** 兩種模式自由切換：
  - 抽籤模式：隨機抽 4 片板子 + 加分點。
  - 手動輸入模式：自己指定要用的 4 片板子，以及每個加分點落在哪一列（用來重現實際比賽抽到的題目）。
- 🧩 **手動擺放**：板子可拖曳放置，會**吸附到格子並預覽落點**（綠色=可放、紅色=超出場地）；拖出場地即移除、左鍵旋轉、右鍵移除。
- 🎞️ **規劃動畫**：規劃完成後，板子會由起點往終點**逐片排列**呈現。
- ⚡ **激進模式**：把「緩衝」從剛放的 2 片放寬成 1 片，通常能找到更少組數的解。
- 📝 **資訊欄**：可填校名/隊名/選手簽名（預設隱藏，需要時開啟，方便截圖留存）。
- 📱 **自適應 + 手機優化**：畫面隨螢幕大小自動縮放；手機上場地可橫向捲動瀏覽。

## 三個組別規則（內建）

| 組別 | 抽板子 | S 板 | 加分點欄位 |
|------|--------|------|------------|
| 國小 | 4 片 | 無 | 第 12、24 欄 |
| 國中 | 4 片 | 每組 1 片（順序不限） | 第 9、18、27 欄 |
| 高中 | 4 片 | 每組 1 片（**必須是該組最後一片**） | 第 9、18、27 欄 |

共同規則：抽到的板子整場循環使用、剛放的板子要先「緩衝」幾片才能再用、可壓板走回頭路、軌道必須經過所有加分點 ◎、最後一片要超過終點線，且終點要再多壓一塊板子。

> ⚠️ 規則以**官方公告為準**。本工具依上方規則連結建置，若官方規則更新，請以官方版本為主（或回報我調整）。

## 操作方式

1. 選**組別**（國小／國中／高中）。
2. 選模式：**抽籤**（隨機題目）或**手動輸入**（重現實際題目）。
3. 按 **自動規劃**，看演算法畫出可行路線與「使用組合」。
4. 想自己試：從下方板子盤**拖曳**板子到場地；左鍵旋轉、右鍵移除、拖出場地移除。
5. 想要更激進的解可開 **激進模式**；想換題目按 **抽籤** 或 **重抽加分點**。

## 技術與部署

- **前端**：React 18 + Vite 5（純前端，無後端、不蒐集任何資料）。
- **規劃核心**：自寫的 A\* 搜尋，位於 [`src/planner/`](src/planner/)。
- **部署**：推送到 `main` → GitHub Actions 自動建置並發布到 GitHub Pages。
- **程式碼保護**：建置時自動對自家程式碼做混淆（[`scripts/obfuscate.mjs`](scripts/obfuscate.mjs)），React 等第三方維持原樣。

### 本機開發

```bash
npm install      # 安裝相依套件
npm run dev      # 本機開發(http://localhost:5173)
npm run build    # 生產建置 + 混淆(輸出到 dist/)
npm run preview  # 預覽建置結果
```
